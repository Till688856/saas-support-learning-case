# SaaS Support Learning Case

This repository contains simple SaaS support case studies created as part of my preparation for remote Technical Support, Product Support and Application Support roles.

The goal is to practice structured troubleshooting:
- understanding the customer issue
- asking the right questions
- reproducing the problem
- investigating possible causes
- documenting the root cause
- writing a clear customer response
- knowing when to escalate to engineering

## Background

My background is in ICT support, application development and business application support.  
This project helps me connect my current experience with SaaS/Product Support roles.

## Included cases

- `01-user-sees-no-data.md` - User can log in, but the dashboard is empty

## Note

This is a learning project. The cases are fictional but realistic.

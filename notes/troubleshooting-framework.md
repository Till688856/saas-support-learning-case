# Troubleshooting Framework

This is a simple framework I use to structure SaaS support cases.

## 1. Understand the issue
First, I try to understand what the customer is actually reporting.

Questions:
- What is not working?
- When did it start?
- Is there an error message?
- Is it affecting one user or multiple users?

## 2. Reproduce or verify
I check if the issue can be reproduced.

Examples:
- Test with another user
- Check browser/session
- Check the same page or feature
- Compare affected and working users

## 3. Check simple causes first
Before escalating, I check common causes.

Examples:
- Filters
- Date range
- Permissions
- Workspace or organization
- Browser cache
- Account status

## 4. Identify root cause
The goal is to find the most likely reason for the issue.

## 5. Communicate clearly
The customer response should be simple, helpful and not blame the user.

## 6. Escalate when needed
Engineering should be involved if there is a bug, data issue, API issue or unclear system behavior.
